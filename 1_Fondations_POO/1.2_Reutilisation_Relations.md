# 1.2 Réutilisation et Relations

## Héritage (Relation "est un" - IS-A) via extends

L'**héritage** permet à une classe (sous-classe) d'hériter des attributs et méthodes d'une autre classe (super-classe). C'est une relation "EST-UN".

```java
public class Animal {
    public void manger() {
        System.out.println("L'animal mange");
    }
}

public class Chien extends Animal {
    public void aboyer() {
        System.out.println("Ouaf!");
    }
}

Chien c = new Chien();
c.manger(); // Hérité de Animal
c.aboyer(); // Propre à Chien
```

## Redéfinition des fonctions de base (via super)

Une sous-classe peut **redéfinir** (override) une méthode de la super-classe. Le mot-clé **super** permet d'accéder aux méthodes de la super-classe.

```java
public class Animal {
    public void crier() {
        System.out.println("Bruit générique");
    }
}

public class Chien extends Animal {
    @Override
    public void crier() {
        super.crier(); // Appel de la méthode parente
        System.out.println("Ouaf!");
    }
}
```

## Composition/Agrégation (Relation "possède un" - HAS-A)

La **composition** (HAS-A) permet à une classe de contenir une instance d'une autre classe. C'est une alternative à l'héritage.

```java
public class Moteur {
    public void demarrer() {
        System.out.println("Moteur démarré");
    }
}

public class Voiture {
    private Moteur moteur; // Composition
    
    public Voiture() {
        this.moteur = new Moteur();
    }
    
    public void demarrer() {
        moteur.demarrer();
    }
}
```

## Interfaces (Déclaration, Implémentation via implements, Hiérarchie via extends)

Une **interface** définit un contrat que les classes doivent respecter. Elle ne contient que des déclarations de méthodes (abstraites).

```java
public interface Animal {
    void crier();
    void manger();
}

public class Chien implements Animal {
    @Override
    public void crier() {
        System.out.println("Ouaf!");
    }
    
    @Override
    public void manger() {
        System.out.println("Mange des croquettes");
    }
}

// Hiérarchie d'interfaces
public interface AnimalTerrestre extends Animal {
    void marcher();
}
```

## Polymorphisme et types d'objets

Le **polymorphisme** permet à une variable de type superclasse de référencer des objets de sous-classes. Les méthodes appropriées sont appelées selon le type réel de l'objet.

```java
Animal a = new Chien();  // Polymorphisme
a.crier();               // Appelle Chien.crier()

a = new Chat();
a.crier();               // Appelle Chat.crier()

// Vérification du type réel
if (a instanceof Chien) {
    Chien c = (Chien) a; // Cast
}
```

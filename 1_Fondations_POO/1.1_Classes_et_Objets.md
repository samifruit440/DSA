# 1.1 Classes et Objets

## Principes de modélisation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation basé sur le concept de "classes" et "objets". Elle permet une meilleure organisation et réutilisation du code.

## Définition d'une classe (attributs et fonctions membres/méthodes)

Une **classe** est un modèle ou un blueprint qui définit la structure et le comportement des objets.

- **Attributs** : Variables qui stockent l'état de l'objet
- **Méthodes** : Fonctions qui définissent le comportement de l'objet

```java
public class Personne {
    // Attributs
    private String nom;
    private int age;
    
    // Méthode
    public void afficher() {
        System.out.println("Nom: " + nom + ", Age: " + age);
    }
}
```

## Un objet est une instanciation d'une classe

Un **objet** est une instance concrète d'une classe. Il possède des valeurs spécifiques pour chaque attribut défini par la classe.

```java
Personne p = new Personne(); // Création d'un objet
```

## Constructeurs et initialisation des objets

Un **constructeur** est une méthode spéciale appelée lors de la création d'un objet. Il permet d'initialiser les attributs.

```java
public class Personne {
    private String nom;
    private int age;
    
    // Constructeur
    public Personne(String nom, int age) {
        this.nom = nom;
        this.age = age;
    }
}
```

## Références et types primitifs en Java

- **Types primitifs** : int, double, boolean, char, etc. (stockés directement en mémoire)
- **Types de référence** : Objets, tableaux, chaînes (pointeurs vers la mémoire)

```java
int x = 5;           // Type primitif
Personne p = new Personne(); // Type de référence
```

## Le principe d'encapsulation (masquage de l'information)

L'**encapsulation** consiste à cacher les détails internes et à fournir une interface publique.

- **private** : Accessible uniquement dans la classe
- **public** : Accessible partout
- **protected** : Accessible dans la classe et les sous-classes

```java
public class Personne {
    private int age;
    
    // Getters et Setters
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age > 0) this.age = age;
    }
}
```

## Passage des paramètres par valeur

En Java, les paramètres sont toujours passés **par valeur**. Pour les types primitifs, une copie de la valeur est passée. Pour les objets, une copie de la référence est passée.

```java
public void modifier(int x) {
    x = 10; // N'affecte pas la variable originale
}

int valeur = 5;
modifier(valeur);
// valeur vaut toujours 5
```

## Membres et méthodes static

Les membres **static** appartiennent à la classe, pas à l'objet. Ils sont partagés par toutes les instances.

```java
public class Compteur {
    private static int count = 0;
    
    public Compteur() {
        count++;
    }
    
    public static int getCount() {
        return count;
    }
}
```

## Surcharge d'opérateurs (notamment dans la classe String)

La **surcharge** permet d'avoir plusieurs méthodes avec le même nom mais des signatures différentes.

```java
// Surcharge du constructeur
public class Personne {
    public Personne() {}
    public Personne(String nom) {}
    public Personne(String nom, int age) {}
}

// Surcharge d'opérateur en Java se fait via des méthodes
String s1 = "Hello";
String s2 = " World";
String s3 = s1 + s2; // Utilise la méthode surchargée concat()
```

# 4.1 Notions de Base

## Utilité des tris

Le tri est une opération fondamentale en informatique. Ses applications incluent :

- **Recherche optimisée** : Les données triées permettent la recherche binaire O(log n)
- **Opérations de base de données** : Regroupement, fusion, jointures
- **Analyse de données** : Détection de doublons, statistiques
- **Affichage** : Listes ordonnées pour les utilisateurs
- **Optimisation d'algorithmes** : Beaucoup d'algorithmes supposent des données triées

## Tri interne vs Tri externe

### Tri Interne

Les données tiennent entièrement en **mémoire RAM**. L'accès à la mémoire est rapide.

- **Avantages** : Rapide, accès aléatoire O(1)
- **Limitations** : Limité par la taille de la RAM
- **Exemples** : QuickSort, MergeSort, HeapSort

### Tri Externe

Les données sont trop volumineuses pour tenir en RAM et résident sur **disque/SSD**.

- **Défi** : Les accès disque sont très coûteux (millisecondes)
- **Stratégie** : Minimiser les I/O disque en traitant des blocs
- **Exemples** : Algorithmes de tri-fusion multi-niveaux, tri par distribution

### Comparaison

| Aspect | Interne | Externe |
|--------|---------|---------|
| Données | En RAM | Sur disque |
| Accès | O(1) en mémoire | O(1) disque, coûteux |
| Algorithmes | QuickSort, MergeSort | MergeSort adapté |
| Limitation | Taille RAM | Volume disque |
